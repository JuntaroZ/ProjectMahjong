概要

MahjongLib は、麻雀の手牌（てはい）解析、和了（ほうら）形判定、役判定、シャンテン計算、和了（ほうら）候補探索、表示用点数計算を行う C++17 ライブラリです。

本ライブラリは mahjong 名前空間に公開 API を持ち、画面表示や入力操作に依存しません。Win32 / GDI+ を使用する画面処理は ProjectMahjong.cpp 側の責務です。

公開 API

EvaluateYaku

YakuResult EvaluateYaku(const std::vector<Tile>& closedTiles, const HandContext& context);
  • 14牌の和了（ほうら）形判定と役判定を行います。
  • 14牌以外の場合は std::invalid_argument を送出します。
  • 役満（やくまん）が成立した場合、通常役は評価結果に含めません。
  • 通常役は、複数の面子（めんつ）分解候補の中から合計翻数（はんすう）が最大になるものを採用します。

EvaluateCurrentYaku

YakuResult EvaluateCurrentYaku(const std::vector<Tile>& closedTiles, const HandContext& context);

現在の14牌に対して、和了（ほうら）形でなくても成立し得る表示用役を評価します。

CalculateShanten

int CalculateShanten(const std::vector<Tile>& tiles);

通常形、七対子（ちーといつ）、国士無双（こくしむそう）のシャンテン数を計算し、最小値を返します。

FindWinningCandidates

std::vector<Tile> FindWinningCandidates(const std::vector<Tile>& hand, const HandContext& context);
  • 14牌の最後の1牌をツモ／ロン牌として扱い、残り13牌に対する和了（ほうら）候補を列挙します。
  • すでに4枚使われている牌は候補に含めません。
  • EvaluateYaku で和了（ほうら）形になる牌を候補として返します。

EvaluateDisplayYaku

std::vector<Yaku> EvaluateDisplayYaku(
    const std::vector<Tile>& hand,
    const HandContext& context,
    const std::vector<Tile>& winningCandidates);
  • 画面表示向けの役一覧を生成します。
  • 現在の14牌が和了（ほうら）形の場合、その和了（ほうら）形の役を返します。
  • 表示用役一覧から 立直 は除外します。
  • 同じ役名は重複して追加しません。

CalculateDisplayYakuScores

std::vector<YakuScore> CalculateDisplayYakuScores(
    const std::vector<Yaku>& yaku,
    const HandContext& context);

表示用の点数文字列を生成します。表示は 翻数（はんすう） / 点数 形式です。

現在の点数は、簡易符を使った子ロン相当の表示点です。親、子、ロン、ツモなどの支払い表記は表示しません。

平和（ぴんふ）の待ち判定

平和（ぴんふ）は以下の場合に成立しません。

  • 待ちが雀頭（じゃんとう）の場合
  • 嵌張（かんちゃん）待ちの場合
  • 辺張（ぺんちゃん）待ちの場合
  • 雀頭（じゃんとう）が三元牌（さんげんぱい）、自風（じかぜ）、場風（ばかぜ）の場合
  • 門前（めんぜん）でない場合

現在の実装では context.winningTile を使用して両面（りゃんめん）待ちかどうかを判定します。

三暗刻（さんあんこう）のロン判定

  • ツモの場合、刻子（こーつ）は暗刻（あんこう）として扱います。
  • ロンの場合、和了牌（ほうらはい）で完成した刻子（こーつ）は明刻（みんこー）扱いとして暗刻（あんこう）数から除外します。

インデックス範囲牌

AnalyzeHand は、牌カウント、通常形の全分解候補、七対子（ちーといつ）形、国士無双（こくしむそう）形を解析します。同一牌が5枚以上ある場合は std::invalid_argument を送出します。

制限事項

  • 現在の公開 API は基本的に閉じた手牌（てはい）ベースです。
  • 鳴き（なき）面子（めんつ）を外部から明示的に渡す API はありません。
  • ドラ、赤ドラ、裏ドラ、場棒、本場は未対応です。
  • 点数計算の符は簡易仕様です。
  • ダブル役満（やくまん）、数え役満（かぞえやくまん）の細かい扱いは未整備です。
  • 四槓子（すーかんつ）は槓（かん）情報を保持するデータ構造がないため未対応です。
  • 流し満貫（ながしまんがん）、人和（れんほう）など、局進行情報が必要な役は未対応です。
  • 役判定は現在実装済みの役に限定されます。

残課題

  • 鳴き（なき）の処理は未対応です。チー、ポン、明槓（みんかん）、暗槓（あんかん）、加槓（かかん）などの面子（めんつ）情報を保持するデータ構造と、それを使った役判定 API の追加が必要です。

使用例

#include "MahjongYaku.h"

#include <iostream>
#include <vector>

int main()
{
    using namespace mahjong;

    std::vector<Tile> hand = {
        Man(1), Man(2), Man(3),
        Pin(1), Pin(2), Pin(3),
        Sou(1), Sou(2), Sou(3),
        Man(7), Man(8), Man(9),
        Honor(1), Honor(1)
    };

    HandContext context;
    context.winningTile = hand.back();
    context.seatWind = Wind::East;
    context.roundWind = Wind::East;
    context.isClosed = true;
    context.isTsumo = false;

    YakuResult result = EvaluateYaku(hand, context);
    if (result.validWinningShape) {
        std::vector<YakuScore> scores = CalculateDisplayYakuScores(result.yaku, context);
        for (const YakuScore& score : scores) {
            std::cout << score.yaku.name << " " << score.text << "\n";
        }
    }
}
